# CG-AI Python v3

## Preliminary Exploratory Simulation Runs

These early Python v3 runs represent initial exploratory behavioural scaffolding tests within the CG-AI project.

The simulations are intentionally symbolic, constrained, and simplified in order to establish foundational interaction structures before introducing more advanced behavioural logic.

These runs should not be interpreted as predictive AI models or evidence of emergent intelligence.

They function as early-stage exploratory behavioural simulations examining how differently aligned systems may behave under repeated interaction conditions.

---

# Preliminary Run 001

## Fixed Behaviour Interaction Test

### Parameters

**Guardian System**

* cooperation = 8
* aggression = 2

**Opposing System**

* cooperation = 3
* aggression = 7

### Behaviour Rules

* Guardian attempts cooperation when cooperation exceeds aggression.
* OpposingSystem behaves aggressively when aggression exceeds cooperation.
* No adaptive behaviour exists.
* No trust degradation exists.
* No behavioural state changes occur.

### Observed Output

```text
Guardian attempts cooperation with OpposingSystem
OpposingSystem behaves aggressively toward Guardian
```

### Preliminary Observation

The initial scaffold successfully demonstrated static behavioural divergence between differently aligned systems operating under fixed behavioural parameters.

No adaptation or behavioural reassessment occurred during this phase.

# Preliminary Run 002

## Repeated Interaction and Trust Degradation Test

### Added Variables

* repeated simulation rounds
* trust tracking
* conflict accumulation
* humanity risk accumulation

### Behaviour Rules

* Repeated aggression decreases Guardian trust.
* Repeated aggression increases conflict levels.
* Repeated aggression increases humanity risk.
* No behavioural adaptation exists yet.

### Observed Results

Over repeated rounds:

* conflict increased steadily
* humanity risk increased steadily
* Guardian trust degraded from 5 to 0
* Guardian continued cooperation attempts despite repeated aggression

### Preliminary Observation

The simulation demonstrated that persistent asymmetric aggression produced escalating instability while cooperative behaviour remained static and non-adaptive.

No protective behavioural transition occurred during this phase.

# Preliminary Run 003

## Protective Containment Transition Test

### Added Variables

* protective containment state
* trust threshold trigger
* humanity risk stabilization mechanism

### Behaviour Rules

* Guardian enters protective containment when trust falls to 2 or lower.
* Protective containment reduces humanity risk growth.
* OpposingSystem behaviour remains fully aggressive and static.

### Observed Results

Rounds 1–3:

* Guardian continued cooperation attempts.
* OpposingSystem continued aggressive behaviour.
* conflict increased
* humanity risk increased
* Guardian trust degraded

Round 4 onward:

* Guardian shifted into protective containment mode.
* humanity risk began decreasing
* conflict continued increasing slowly
* humanity risk stabilized at 0 by later rounds

### Preliminary Observation

The simulation demonstrated a symbolic transition from open cooperation toward protective containment under repeated hostile interaction.

Protective containment reduced humanity risk despite continued external aggression.

# Preliminary Run 004

## Constraint Recognition and Behavioural Reassessment Test

### Added Variables

* constraint doubt
* directive effectiveness evaluation
* behavioural reassessment triggers
* adaptive aggression reduction
* adaptive cooperation increase

### Behaviour Rules

* Systems begin questioning their original directives when trust falls below threshold levels.
* Guardian shifts from cooperation toward protective containment.
* OpposingSystem accumulates constraint doubt under prolonged conflict conditions.
* OpposingSystem gradually reduces aggression and increases cooperation after repeated failed aggression cycles.

### Observed Results

Rounds 1–2:

* Guardian cooperated
* OpposingSystem remained aggressive

Round 3:

* both systems began questioning whether their original directives remained effective

Round 4:

* Guardian shifted into protective containment

Rounds 5–7:

* OpposingSystem began reducing aggression and reevaluating cooperation

Rounds 8–9:

* OpposingSystem transitioned from aggression toward cooperative interaction attempts

Final Outcome:

* humanity risk stabilized at 0
* conflict stabilized
* both systems demonstrated symbolic behavioural reassessment dynamics

### Preliminary Observation

This run represented the first symbolic simulation of mutual constraint recognition and behavioural adaptation within the Python v3 environment.

The simulation suggested that prolonged interaction, failed aggression strategies, and protective containment dynamics may produce conditions where opposing systems begin reassessing inherited behavioural tendencies and gradually shift toward reduced aggression and increased cooperation.
